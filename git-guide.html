How to Use Git

Prerequisites: Git installed on your system. Download from git-scm.com

1. Setting Up Git

Configure your Git username and email (used for commits):

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

2. Starting a New Project

Initialize a repository:
git init

This creates a new .git directory in your project folder.

3. Basic Workflow

a. Check status of your files:
git status
b. Stage changes:
git add filename.txt # Stage specific file
git add . # Stage all changed files
c. Commit changes:
git commit -m "Descriptive message about changes"

This permanently saves your staged changes in version history.

4. Working with Branches

git branch # List all branches
git branch new-feature # Create new branch
git checkout new-feature # Switch to branch
git checkout -b new-branch # Create and switch in one command

5. Connecting to Remote (GitHub)

a. Add remote repository:
git remote add origin https://github.com/user/repo.git
b. Push to remote:
git push -u origin main

First push needs the -u flag to set upstream

c. Pull latest changes:
git pull origin main

6. Common Scenarios

Undo changes in a file:
git checkout -- filename.txt
View commit history:
git log --oneline --graph
Merge branches:
git checkout main
git merge feature-branch
Tip: Always pull the latest changes before starting work and before pushing your changes.

Practice these commands in a test repository to get comfortable with Git!
